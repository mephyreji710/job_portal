TalentBridge

Job Portal with Intelligent Resume Screening

A full-stack web application that connects job seekers with recruiters, featuring AI-powered resume screening, automated matching, and an integrated resume builder.

Module Overview — 10 Django Applications
Each module is a self-contained Django application responsible for a specific domain of the system.

Module 1 — accounts

3 Models
Handles user authentication, registration, login (email-based), and role management. Provides three roles: Admin, Recruiter, and Job Seeker. Includes password reset via email and per-role dashboard routing.
User RecruiterProfile JobSeekerProfile (basic)

Module 2 — jobseeker

5 Models
Full job seeker profile management. Stores detailed professional information including skills with proficiency levels, education history, work experience with duration calculations, and certifications. Includes profile completeness scoring.
JobSeekerProfile Skill Education Experience Certification

Module 3 — resume

4 Models
Two-part module: (1) Resume Upload & Parsing — accepts PDF/DOCX files, extracts text using pdfplumber and python-docx, identifies skills, keywords, education level, and experience years. (2) Resume Builder — creates formatted resumes from profile data with 5 ATS-optimised templates, custom colours, fonts, and section ordering.
Resume ParsedResume ExtractedKeyword BuiltResume

Module 4 — recruiter

1 Model
Company/Recruiter panel for managing the company profile, branding (logo, tagline, description), industry, size, and team. Allows adding HR team members and provides a company completeness indicator.
HRMember

Module 5 — jobs

2 Models
Job posting management. Recruiters create and manage job listings with required skills, experience level, education requirements, salary range, and deadline. Job seekers browse the public job board and can save jobs. Supports draft/active/paused/closed status lifecycle.
JobPost SavedJob

Module 6 — screening

1 Model
Intelligent resume screening engine. Computes a weighted match score between a candidate's profile and a job post: Skills 40%, Experience 25%, Education 15%, Keywords 20%. Stores matched/missing skills and keywords. Recruiters see a ranked candidate list per job.
MatchScore

Module 7 — applications

1 Model
Job application lifecycle management. Job seekers submit applications with cover letter, skills summary, and attached resume. Recruiters review and update status through: Pending → Reviewed → Shortlisted → Rejected / Hired. One application per user per job enforced.
Application

Module 8 — feedback

1 Model
Bidirectional feedback system. After an application reaches a terminal status, both the candidate and the company can leave a 1–5 star rating with a comment. Supports anonymous mode. Ensures one feedback per direction per application.
Feedback

Module 9 — interviews

1 Model
Interview scheduling and management. Linked one-to-one with an application. Recruiters schedule video, phone, or on-site interviews with date/time, duration, meeting link or venue. Status lifecycle: Scheduled → Confirmed → Completed / Cancelled. Candidate notifications triggered on changes.
Interview

Module 10 — notifications

1 Model
In-app notification system. Generates notifications for all key events: application received, status changes (shortlisted, rejected, hired), interview scheduled/confirmed/cancelled. Each notification links to the relevant page, supports read/unread state, and displays an icon and colour based on type.
Notification

11 HRMember FK RecruiterProfile companyCASCADE One company can have many HR team members
12 JobPost FK RecruiterProfile recruiterCASCADE One recruiter/company can post many jobs
13 SavedJob FK User + JobPost user, jobCASCADE Bridge table — job seeker saves many jobs (M:M via bridge)
14 MatchScore FK JobPost + User job, candidateCASCADE AI match score between a candidate and a job (unique pair)

16 Feedback FK Application + User application, from_userCASCADE Bidirectional post-process feedback linked to an application
